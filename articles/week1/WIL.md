### 2. 데이터를 train과 test로 분리하기
- train data(훈련 데이터) : 모델을 학습시킬 때 사용하는 데이터(모델 가중치들을 업데이트할 때 사용한다.)
- test data(테스트 데이터) : 훈련 데이터로 학습시킨 모델을 테스트할 때 사용하는 데이터

> 과적합(overfitting) : 모델이 훈련데이터에 맞춰져서 테스트 데이터로 테스트할 때, 성능이 낮아지는 현상

과적합을 항상 주의해야 한다.
### 3. 데이터에서 Label(정답)을 분리 및 전처리하기
데이터가 현재 id, document, label 순서대로 정리되어있으므로, label만 따로 분리하여 y(정답) 데이터를 만들어준 후 전처리 해야한다.

이를 통해 지도학습을 진행할 수 있다.
- 지도학습(Supervised Learning) : Label(정답, y)이 있는 Data(입력, X)로 모델을 학습시키는 것
- 지도학습과 비지도 학습의 차이점 : 비지도학습은 Label이 존재하지 않고, X에 내재된 패턴, 구조, 특징들을 파악하는데 초점을 맞춘다. 대표적인 학습 방식으로 클러스터링, 차원 축소 등이 존재

그 후, 텍스트 데이터를 숫자 시퀀스로 변환하는 **전처리**과정을 진행한다.
여기서 keas.layers.**TextVectorization**를 사용하여 Input을 **벡터**로 변환한 후, **패딩**을 진행한다.
- TextVectorization : 입력 문자를 받아, 소문자로 변환 & 구두점 제거하고 띄어쓰기를 기준으로 단어를 나누기&각 단어를 정수 인덱스로 변환
- 벡터화가 필요한 이유 : 컴퓨터는 문자열을 바로 이해하지 못하기 때문에, 문자열에 담긴 정보를 정량적으로 표현해야 한다.
- 패딩(Padding) : 입력을 같은 크기로 만드는 역할 / 사전에 정한 단어 사전의 크기보다 Input이 작을경우 빈 공간을 0으로 채워준다.
